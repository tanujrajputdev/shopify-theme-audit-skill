## Shopify Theme Audit Report

**Theme:** Atelier (Online Store 1.0, heavily customized)
**Date:** 2026-04-22

**Score:** 31/100 — Grade: F

This theme predates Online Store 2.0. Many findings reflect that age — they are not bugs in original authoring, just an accumulation of Shopify API deprecations. The store should plan a migration to a 2.0 theme rather than patching this one indefinitely. This audit identifies what to fix immediately to stabilise revenue until migration completes.

### Critical Issues (6) — Fix these immediately

**C1: Render-blocking scripts in head**
File: `layout/theme.liquid`, Lines: 38, 42, 47, 51, 55
Problem: Five separate first-party `<script src>` tags in `<head>` without `defer` or `async`. jQuery 1.11, the theme's `option_selection.js`, and three vendor bundles all load synchronously.
Impact: ~1.2s of render-blocking time on mid-range mobile. First Contentful Paint pushed past 3.5s.
Fix:
```liquid
<script src="{{ 'jquery.min.js' | asset_url }}" defer></script>
<script src="{{ 'theme.js' | asset_url }}" defer></script>
<script src="{{ 'option_selection.js' | shopify_asset_url }}" defer></script>
```

---

**C2: Deprecated `img_url` filter used throughout**
Files: `snippets/product-card.liquid:14`, `snippets/product-grid-item.liquid:21,29`, `sections/featured-collection.liquid:78`, `sections/main-product.liquid:112,134,168`, `templates/article.liquid:43`, plus 11 other locations
Problem: `img_url` is deprecated and does not serve WebP. Every product image is served as JPEG even to browsers that support WebP.
Impact: ~30-40% larger image payloads than necessary. Lighthouse Performance penalty.
Fix: See `before-after.md` § C2 for the complete replacement pattern. Sweep all 17 occurrences.

---

**C3: No lazy loading on product grid**
Files: `snippets/product-grid-item.liquid:21`, `sections/featured-collection.liquid:78`
Problem: Collection pages render 24 product images, all loaded eagerly.
Impact: ~2-4MB of unused initial bytes on first paint. Mobile LCP delayed by 1.5-3s.
Fix:
```liquid
<img src="{{ product.featured_image | image_url: width: 400 }}"
     loading="{% if forloop.first %}eager{% else %}lazy{% endif %}"
     width="{{ product.featured_image.width }}"
     height="{{ product.featured_image.height }}"
     alt="{{ product.featured_image.alt | default: product.title | escape }}">
```

---

**C5: No skip-to-content link**
File: `layout/theme.liquid`, Line: 64
Problem: No keyboard-accessible skip link as the first focusable element after `<body>`.
Impact: WCAG 2.1 AA violation. Keyboard users and screen reader users must tab through 40+ navigation items to reach main content.
Fix: See `before-after.md` § C5.

---

**C6: Images missing dimensions site-wide**
Files: 14 separate locations across `sections/` and `snippets/`
Problem: `<img>` tags without explicit `width` and `height`. The header logo is the worst offender — visible jump on every page load.
Impact: CLS score above 0.25 on most templates. Google Core Web Vitals failing.
Fix: See `before-after.md` § C6. Set intrinsic dimensions and let the browser scale.

---

**S-C5: No Product JSON-LD on PDP**
File: `templates/product.liquid`
Problem: No structured data emitted for products. Rich snippets unavailable in Google Search. AI engines cannot reliably extract price, availability, or brand.
Impact: Lost rich snippets in classic search, lost machine-readability for AEO/GEO.
Fix: See `before-after.md` § M4.

### High Priority (9)

**H1: Remaining inline scripts**
File: `templates/product.liquid:178-220`
Problem: 42 lines of inline variant-picker JavaScript embedded in the Liquid template. Runs synchronously on every PDP load.
Fix: Extract to `assets/product-form.js`, load with `defer`.

**H2: No WebP**
Cascading effect of C2. Once C2 is fixed, this resolves.

**H3: No preload for hero image**
File: `templates/index.liquid:8` (hero hardcoded into template)
Problem: The homepage hero loads at request time but has no `<link rel="preload">` hint.
Fix: See `before-after.md` § H3.

**H4: `theme.scss.liquid` compiled into 184KB CSS**
File: `assets/theme.scss.liquid`
Problem: Single CSS file containing styles for every section type, including types not used on the current template.
Recommendation: Defer non-critical CSS, or migrate to per-section CSS during the 2.0 migration.

**H5: No focus trap on quick-view modal**
File: `assets/quickview.js`
Problem: Modal opens via a quick-view button but keyboard focus is not trapped. Tab key escapes behind the modal to underlying page elements.

**S-H1: Header logo wrapped in `<h1>` on every page**
File: `sections/header.liquid:34`
Problem: Logo emits `<h1>` globally. Every page's content `<h1>` competes with it, splitting on-page topical relevance.
Fix:
```liquid
{% if template == 'index' %}
  <h1 class="site-logo">…</h1>
{% else %}
  <div class="site-logo">…</div>
{% endif %}
```

**S-H2: Heading hierarchy broken**
File: `templates/page.liquid:18`
Problem: Page template skips from `<h1>` directly to `<h4>` for "About us" pages.

**AEO-C1: FAQ accordion present, no FAQPage schema**
File: `sections/faq.liquid:1-60`
Problem: 8 FAQ items rendered as `<details>` elements with no JSON-LD.
Fix: See `before-after.md` § AEO-C1.

**GEO-C1: robots.txt does not address AI crawlers**
File: No `templates/robots.txt.liquid` exists — theme uses Shopify default
Problem: Shopify default robots.txt allows all bots, which is fine. But there is no explicit signal of intent. The merchant should make an explicit choice and document it.
Fix: Add a `templates/robots.txt.liquid` per `before-after.md` § GEO-C1.

### Recommended Fix Order

1. Add lazy loading + width/height to all product grid images (C3 + C6) — single sweep, biggest single improvement to LCP and CLS
2. Defer all `<script src>` tags in `theme.liquid` (C1 + H1) — biggest single FCP improvement
3. Replace `img_url` with `image_url` across the 17 files (C2) — unlocks WebP automatically
4. Add Product JSON-LD on PDP (S-C5 / M4) — one snippet, immediate SERP impact
5. Add FAQ schema to existing FAQ section (AEO-C1) — already have the content, just add the schema
6. Add skip-to-content link (C5) — one block in `theme.liquid`, accessibility compliance unlocked
7. Add hero preload hint (H3)
8. Add Organization schema on homepage (GEO-H1)
9. Plan the Online Store 2.0 migration — this theme has structural debt that no amount of patching resolves
