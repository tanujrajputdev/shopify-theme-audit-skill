### Critical Issues (1) — Fix these immediately

**C6: Images missing height attribute (CLS)**

Multiple `<img>` tags set only `width=` with no `height=`, so the browser cannot reserve vertical space before the image loads. This causes layout shift, which Google penalizes in Core Web Vitals. The above-the-fold logo is the worst offender.

Files:
- `sections/header.liquid:192` and `:237` — hardcoded logo fallback, above-the-fold
- `sections/hero.liquid:18`, `:44` — fallback hero images
- `sections/benefits.liquid:23`
- `sections/ugc-strip.liquid:26`, `:50`
- `sections/problem-solution.liquid:18`, `:57`
- `sections/pdp-crosssell.liquid:93`, `:109`

Impact: Visible page jump on first load. Degrades LCP and CLS Lighthouse scores. The header logo shifts when it loads.

Fix:
```liquid
{%- comment -%} For asset images with a known square ratio {%- endcomment -%}
<img
  src="{{ 'product-image.png' | asset_url }}"
  alt="{{ product.title | escape }}"
  loading="lazy"
  width="600"
  height="600">

{%- comment -%} For the above-fold header logo {%- endcomment -%}
<img
  src="{{ settings.logo | image_url: width: 360 }}"
  alt="{{ shop.name }}"
  width="180"
  height="60"
  fetchpriority="high">
```

### High Priority (3)

**H3: No `<link rel="preload">` for the LCP hero image**

File: `layout/theme.liquid` (head), `sections/hero.liquid:6–17`

The hero uses `loading="eager"` and `fetchpriority="high"` (good), but the browser still discovers the image late in the HTML parsing waterfall. A preload hint surfaces it to the network earlier.

Impact: LCP improves by 300–800ms on mobile 4G.

Fix — add to `layout/theme.liquid` inside `<head>`:
```liquid
{% if template == 'index' %}
  {% for section in template.sections %}
    {% if section.type == 'hero' and section.settings.image != blank %}
      <link
        rel="preload"
        as="image"
        href="{{ section.settings.image | image_url: width: 1400 }}"
        imagesrcset="{{ section.settings.image | image_url: width: 800 }} 800w,
                     {{ section.settings.image | image_url: width: 1400 }} 1400w"
        imagesizes="60vw">
    {% endif %}
  {% endfor %}
{% endif %}
```

---

**H-custom: Render-blocking third-party stylesheets in `<head>`**

File: `layout/theme.liquid:20–21`
```html
<link href="https://fonts.googleapis.com/css2?family=DM+Sans:wght@400;500&display=swap" rel="stylesheet">
<link rel="stylesheet" href="https://cdn.hugeicons.com/font/hgi-stroke-rounded.css">
```

Both block first paint. Google Fonts is redundant — Shopify's native `font_face` filter already loads the body and heading fonts. The icon CDN adds an uncontrolled external dependency.

Impact: 100–400ms blocking time on slow connections.

Fix:
```html
<!-- Replace Google Fonts link with Shopify's font system (already in use for other fonts) -->
<!-- Remove the <link> tag entirely — use font_face for DM Sans instead -->

<!-- Lazy-load the icon stylesheet -->
<link
  rel="stylesheet"
  href="https://cdn.hugeicons.com/font/hgi-stroke-rounded.css"
  media="print"
  onload="this.media='all'">
<noscript>
  <link rel="stylesheet" href="https://cdn.hugeicons.com/font/hgi-stroke-rounded.css">
</noscript>
```

**H4: Global CSS payload at the upper limit**

File: `assets/base.css` — 81KB loaded on every page via `layout/theme.liquid:273`

At the 80KB threshold. Combined with theme-specific CSS (10KB) and font/icon CSS, total blocking CSS exceeds 100KB on mobile.

Fix: Several per-template splits already exist (`pdp.css`, `component-card.css`). Audit `base.css` for selectors only used on a single template and move them to the appropriate file, loaded via `section_tag` only where needed.

### Medium Priority (2)

**M3: Inline hardcoded brand colors**

File: `snippets/cart-drawer.liquid:293`, `:473`
```html
<p style="font-size:13px;color:#9E8E84;">...</p>
<div style="...color:#1D9E75;">...</div>
```

Bypasses the theme's CSS custom properties. The merchant cannot recolor these elements from the theme editor.

Fix: Move to `assets/component-cart-drawer.css` using:
```css
.cart-upsell-text {
  font-size: 13px;
  color: rgba(var(--color-foreground), 0.6);
}
```

**M5: No breadcrumb navigation or BreadcrumbList schema**

Search across all files returned zero results for `breadcrumb` or `BreadcrumbList`. Product and collection templates have no breadcrumb markup.

Impact: Loss of breadcrumb rich snippets in Google SERPs.

Fix — create `snippets/breadcrumbs.liquid`:
```liquid
{% comment %}
  Renders breadcrumb navigation with JSON-LD BreadcrumbList schema.
  Usage: {% render 'breadcrumbs' %}
{% endcomment %}

<nav class="breadcrumbs" aria-label="{{ 'general.breadcrumbs.label' | t }}">
  <ol>
    <li><a href="{{ routes.root_url }}">{{ 'general.breadcrumbs.home' | t }}</a></li>
    {% if collection %}
      <li><a href="{{ collection.url }}">{{ collection.title }}</a></li>
    {% endif %}
    {% if product %}
      <li aria-current="page">{{ product.title }}</li>
    {% endif %}
  </ol>
</nav>

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": {{ shop.name | json }},
      "item": {{ request.origin | json }}
    }
    {% if collection %},{
      "@type": "ListItem",
      "position": 2,
      "name": {{ collection.title | json }},
      "item": {{ request.origin | append: collection.url | json }}
    }{% endif %}
    {% if product %},{
      "@type": "ListItem",
      "position": {% if collection %}3{% else %}2{% endif %},
      "name": {{ product.title | json }},
      "item": {{ request.origin | append: product.url | json }}
    }{% endif %}
  ]
}
</script>
```

Render in `sections/main-product.liquid` and `sections/main-collection-product-grid.liquid`:
```liquid
{% render 'breadcrumbs' %}
```
